# Build Prompt — WorkGPT Admin (Frontend cho Multi-Channel Bot Backend)

> Đây là PROMPT để đưa cho AI coding assistant (Claude / Cursor / v0 / Lovable) dựng dashboard FE cho backend NestJS đã có sẵn.
> Đọc nguyên file này và làm theo. Không bỏ bước. Không thêm scope ngoài yêu cầu.
> Backend spec ở `plan.md` + `README.md` cùng repo.

---

## 1. Goal

Xây dashboard quản trị **Admin** cho customer login vào, quản lý nhiều bot Zalo (Telegram tương lai), gán prompt + tài liệu RAG, theo dõi conversation realtime, và chat thay bot. Mục tiêu:

- **Modern · Polished · WOW**: cảm giác như Linear / Notion / Vercel, không phải "admin template thập niên 2015".
- **UX-first**: empty state, loading skeleton, optimistic update, error inline. Người dùng không gặp spinner trắng vô tận.
- **Retention**: onboarding mượt, quick-action visible, quota warning rõ ràng, milestone vui khi đạt mốc đầu tiên.
- **Realtime**: tin nhắn pop lên ngay (Socket.IO), trạng thái bot live (online / expired), QR login push thẳng qua socket.

## 3. Backend API hiện có (tham chiếu, không tự sửa)

Base URL: `http://localhost:3000`
Auth: `Authorization: Bearer <JWT>` (JWT lấy từ `/auth/login`, decode chứa `customerId`).
Swagger interactive: `GET /docs`.
Error envelope đồng nhất: `{ ok:false, error:{ code, message, requestId, detail? } }`.

### Endpoints

```
POST   /auth/login                            → { accessToken }

GET    /bots                                  list bots
GET    /bots/:id                              detail + counts + quota + lastMessageAt
POST   /bots
PATCH  /bots/:id                              update name/promptId/settings/quota
DELETE /bots/:id

GET    /bots/:id/documents                    list attached docs
POST   /bots/:id/documents                    { documentIds: number[] }  (quota-checked)
DELETE /bots/:id/documents/:docId             detach

GET    /conversations?botId=&limit=&cursor=   list conversations of bot, cursor by id
GET    /conversations/:id                     detail + participants
GET    /conversations/:id/messages?cursor=&limit=  chat history (newest first, cursor by msg id)
POST   /conversations/:id/read                reset unread

POST   /messages/send                         { botId, threadId, threadType, text?, attachments? }

POST   /channels/zalo/login                   → { sessionId, hint:{kind:'qr', data:{sessionId}} }
POST   /channels/zalo/logout/:botId

POST   /documents                             { customerId, title, rawText, source?, mimeType? }
POST   /documents/upload                      multipart: file + (title?)
POST   /documents/import-url                  { url, title? }
GET    /documents
DELETE /documents/:id
POST   /documents/:id/reembed

GET    /prompts
POST   /prompts                               { name, template, variables? }
PATCH  /prompts/:id                           tăng version tự động khi template đổi
DELETE /prompts/:id

GET    /health/live  /health/ready            (dùng cho infra, không phải UI)
```

### Socket.IO events

Namespace `/realtime`. Auth qua handshake `auth: { token: '<JWT>' }`. FE join 2 loại room:

- `customer:{customerId}` — auto-join khi handshake OK.
- `session:{sessionId}` — emit `'session:subscribe'` `{ sessionId }` ngay sau khi nhận `sessionId` từ `POST /channels/zalo/login`.

Server emit:

| Event | Payload | Khi nào |
|---|---|---|
| `message:new` | `{conversationId, messageId, direction:'in', text, attachments, senderExternalId, ts}` | Tin nhắn vừa vào |
| `message:sent` | `{conversationId, messageId, direction:'out', text, attachments, ts}` | Bot vừa gửi xong (auto reply hoặc manual) |
| `bot:status` | `{botId, from, to, reason}` | Bot đổi trạng thái (vd. active → expired) |
| `document:status` | `{documentId, from, to, error?}` | Doc xong embed hoặc fail |
| `qr:image` | `{image: 'data:image/png;base64,...'}` | QR Zalo render |
| `qr:scanned` | `{}` | User vừa quét QR |
| `qr:success` | `{}` | Login OK |
| `qr:failed` | `{reason}` | QR expired/declined |

### Quota (CHÚ Ý — UX phải xử lý)

- Mỗi bot có `requestQuota: 30`, `documentQuota: 2` mặc định.
- Khi auto-reply: backend silent skip (UI không biết → cần hiện rõ remaining ở bot detail).
- Khi manual send vượt quota: `402 Payment Required`, body `{error:{code:"QUOTA_REQUEST_EXCEEDED", detail:{used,limit,botId}}}` — FE phải catch và show modal upgrade.
- Khi attach doc vượt: `402 QUOTA_DOCUMENT_EXCEEDED` — modal upgrade.

## 5. Design system

### 5.1. Color palette (CSS variables — light + dark)

Light mode (default Tailwind neutral + accent):
```css
--background: 0 0% 100%;
--foreground: 240 10% 4%;
--card: 0 0% 100%;
--muted: 240 5% 96%;
--muted-foreground: 240 4% 46%;
--border: 240 6% 90%;
--primary: 240 78% 60%;        /* indigo-violet, "WorkGPT brand" */
--primary-foreground: 0 0% 100%;
--success: 142 70% 45%;
--warning: 38 92% 50%;
--destructive: 0 84% 60%;
--ring: 240 78% 60%;
```

Dark mode (đây là **default** — admin dashboard ban đêm cảm giác premium hơn):
```css
--background: 240 10% 5%;
--foreground: 0 0% 98%;
--card: 240 8% 8%;
--muted: 240 6% 12%;
--muted-foreground: 240 5% 64%;
--border: 240 6% 16%;
--primary: 240 90% 70%;
```

**Brand accent**: gradient `from-indigo-500 via-violet-500 to-fuchsia-500` cho hero CTA, logo, primary button background trên dark mode.

### 5.2. Typography

- Font: **Inter Variable** (sans), **JetBrains Mono Variable** (mono cho prompt editor + code).
- Scale Tailwind mặc định (text-xs → text-4xl).
- Page heading: `text-2xl font-semibold tracking-tight`.
- Section heading: `text-base font-medium`.
- Body: `text-sm leading-6`.

### 5.3. Spacing & radius

- Card padding: `p-5` hoặc `p-6`.
- Section gap: `gap-6`.
- Border radius: `rounded-xl` cho card, `rounded-lg` cho input/button, `rounded-full` cho avatar/badge.

### 5.4. Elevation

- Card mặc định: `border` (không shadow) — clean.
- Hover card list: thêm `shadow-sm` + `border-primary/20` transition.
- Modal/sheet: `shadow-2xl`.

### 5.5. Motion

- Page transition: framer-motion `initial={{opacity:0, y:8}} animate={{opacity:1, y:0}} transition={{duration:0.2, ease:'easeOut'}}`.
- Hover card: `hover:scale-[1.01]` + `transition-transform duration-200`.
- Bubble enter: spring `y:10 → 0` 200ms.
- Tránh > 300ms — admin phải feel nhanh, không phải landing.

- **Middle**: conversation list với search, filter (all/unread/group/1-1). Item gồm avatar (DiceBear nếu không có), name, last msg preview (truncate), time (relative), unread badge.
- **Right**: chat panel.
  - Header: thread name + participant count + action menu (mark read, archive, view details).
  - Messages: virtualized list (sử dụng `@tanstack/react-virtual`). Bubble in = trái xám, out = phải gradient brand. Time chỉ hiện khi gap > 5 phút.

  - **Optimistic update**: tin nhắn gửi đi xuất hiện ngay với state "sending" (clock icon mờ), khi server confirm thì đổi sang "sent" (check icon). Fail → red với retry button.
  - **"Bot đang trả lời..."** indicator: 3 chấm bouncing — tạm thời show trong 5s sau khi user gửi (backend chưa có signal real, có thể remove sau).

### 6.7. `/bots/:id/documents` + `/documents`

- Grid card hoặc table toggle.
- Mỗi item: title, source (filename/URL), mime icon, status badge (pending có spinner, embedded có check xanh, failed có x đỏ + tooltip lỗi), createdAt, count chunks (nếu API trả).
- Action: Reembed, Delete, Detach (nếu trong bot view).
- **Upload zone** trên đầu trang: drop zone full-width với border dashed, hover state nổi bật.
  - Drag file vào → border đổi sang accent + scale lên nhẹ.
  - Multi-file queue với progress bar mỗi file.
  - Sau upload xong → toast "Đang nhúng tài liệu...", item xuất hiện status `pending`.
  - Socket `document:status` → update badge `embedded` real-time.
- **Import URL** button → mở dialog với input URL + title (optional). Validate là Google Docs/Sheets link. Có example placeholder.
- Quota: nếu attach mà 402 → mở `QuotaExceededModal` (CTA: "Upgrade plan" — placeholder anchor `mailto:` hoặc link Stripe sau).

### 6.8. `/prompts` + `/prompts/:id`

- List: card mỗi prompt với name, version badge, isActive switch, lastUpdated, snippet 2 dòng từ template.
- Detail/edit:
  - **PromptEditor** dùng Shiki để highlight `{{variable}}` tokens.
  - Sidebar phải hiển thị danh sách biến used + cho phép preview với mock data.
  - **Test panel** dưới editor: input mock JSON `{bot_name:"X", rag_context:"...", user_message:"..."}` → render preview ngay (gọi 1 endpoint mới nếu cần, hoặc làm pure client-side regex).
  - Save → tăng version tự động (backend lo).

### 6.9. `/bots/:id/settings`

3 section card:

1. **General**: name, avatar (upload, preview), default promptId (select).
2. **LLM** (sliders + numeric inputs):
   - Model (select: gpt-4o-mini default, gpt-4o, …)
   - Temperature (slider 0–2, step 0.1)
   - Max tokens (slider 1–4096)
   - Top-p (slider 0–1)
   - Frequency / presence penalty (slider -2–2)
   - Mỗi slider có "Reset to default" link nhỏ.
3. **Quota**: hiển thị usage hiện tại (ring) + 2 input cho `requestQuota` và `documentQuota` (admin override). Có warning text "Tăng quota sẽ áp dụng ngay cho bot này".
4. **Danger zone** (card border red): Reset request counter, Delete bot — confirm dialog.

### 6.10. Global components

**Sidebar** (left, collapsible):
- Logo + name on top.
- Nav items với icon + label (collapse → icon only).
- Active route highlight với accent bar bên trái.
- Bottom: avatar + name + customerId + theme toggle + logout.

## 7. UX behaviors quan trọng

### 7.1. Optimistic updates everywhere

- Send message → bubble xuất hiện ngay với state `sending`, server confirm → `sent`, fail → `failed` + retry button.
- Toggle prompt active → switch animate ngay, server fail → revert + toast.
- Delete bot → remove khỏi list ngay, server fail → restore + toast (sử dụng TanStack Query `onMutate`).

### 7.2. Loading states

- **KHÔNG** dùng spinner toàn trang. Luôn dùng **skeleton** match shape thực tế (BotCard skeleton, MessageBubble skeleton).
- Inline action loading: button disable + spinner nhỏ trong button.

### 7.3. Empty states

Mọi list rỗng phải có:
- Illustration vector (lucide icon to + gradient bg) HOẶC ascii art friendly.
- Heading "Chưa có {entity}".
- 1 dòng giải thích.
- CTA button primary.

Ví dụ bots list rỗng:
```
            🤖
   Chưa có bot nào
   Tạo bot Zalo đầu tiên để bắt đầu nhận tin nhắn tự động
   [   + Tạo bot mới   ]
```

### 7.4. Error states

- Network fail: trang error với illustration + button "Thử lại" + ghi `requestId` nhỏ (copy được) để báo support.
- Form error: inline dưới field, đỏ, không alert popup.
- 402 quota: modal đẹp với progress ring 100% + 2 button "Upgrade" / "Reset trial" (admin-only).

### 7.5. Realtime polish

- Tin nhắn mới khi đang ở conversation khác → notification bell tăng + toast unobtrusive "Tin mới từ Alice trong Bot A" với button "Xem".
- Conversation đang mở: tin mới slide in từ dưới + scroll to bottom nếu user đã ở bottom (KHÔNG auto-scroll nếu user đã scroll lên đọc lịch sử).
- Connection lost: banner top "Đang mất kết nối, thử lại trong 5s..." + auto-reconnect socket.

### 7.6. Onboarding & retention

- First-time login: dashboard hiện checklist 4 bước (xem §6.2).
- Sau khi hoàn thành bước cuối → confetti + modal "🎉 Setup xong! Bot của bạn đã sẵn sàng" với 3 link quick action.
- **Milestone toasts**: tin nhắn thứ 10 / 50 / 100 → toast vui (vd. "🎯 Bot của bạn đã trả lời 50 tin!").
- **Bot expired**: socket `bot:status` to=`expired` → red banner top "Cookie Zalo hết hạn, đăng nhập lại" + button "Re-login" (mở QR modal).

### 7.7. Keyboard shortcuts

- `Cmd+K` Command palette.
- `g b` → Bots, `g c` → Conversations, `g d` → Documents, `g p` → Prompts (Vim-style nav).
- `n` trong chat → focus composer.
- `Esc` → close modal / clear search.
- `?` → show shortcuts cheatsheet modal.

### 7.8. Accessibility

- Tất cả icon button có `aria-label`.
- Focus ring rõ ràng (Tailwind `focus-visible:ring-2 ring-ring`).
- Sidebar nav `role="navigation"`, message list `role="log" aria-live="polite"`.
- Color contrast AA tối thiểu (dark mode primary text trên bg phải ≥ 4.5:1).
- Reduce motion: respect `prefers-reduced-motion` → disable parallax/spring.

## 8. API client setup

### 8.1. `lib/api.ts`

```ts
import { useAuthStore } from '@/stores/auth.store';

const BASE = import.meta.env.VITE_API_URL ?? 'http://localhost:3000';

export class ApiError extends Error {
  constructor(
    public code: string,
    message: string,
    public status: number,
    public requestId?: string,
    public detail?: unknown,
  ) { super(message); }
}

export async function api<T>(
  path: string,
  init: RequestInit = {},
): Promise<T> {
  const token = useAuthStore.getState().token;
  const res = await fetch(`${BASE}${path}`, {
    ...init,
    headers: {
      'Content-Type': 'application/json',
      ...(token && { Authorization: `Bearer ${token}` }),
      ...init.headers,
    },
  });
  if (!res.ok) {
    const body = await res.json().catch(() => ({}));
    throw new ApiError(
      body.error?.code ?? 'UNKNOWN',
      body.error?.message ?? res.statusText,
      res.status,
      body.error?.requestId,
      body.error?.detail,
    );
  }
  if (res.status === 204) return undefined as T;
  return res.json();
}
```

### 8.2. Global error handling

```ts
// providers
<QueryClientProvider client={new QueryClient({
  defaultOptions: {
    queries: { retry: 1, staleTime: 30_000 },
    mutations: {
      onError: (err) => {
        if (err instanceof ApiError) {
          if (err.status === 401) { /* auto-logout */ }
          else if (err.code.startsWith('QUOTA_')) { openQuotaModal(err); }
          else { toast.error(err.message); }
        }
      },
    },
  },
})}>
```

### 8.3. Socket setup

```ts
// lib/socket.ts
import { io, Socket } from 'socket.io-client';
let socket: Socket | null = null;
export function getSocket(token: string): Socket {
  if (socket?.connected) return socket;
  socket = io(`${import.meta.env.VITE_API_URL}/realtime`, {
    auth: { token },
    transports: ['websocket'],
    reconnection: true,
    reconnectionDelay: 1000,
  });
  return socket;
}
```

```ts
// hooks/useRealtimeMessage.ts
export function useRealtimeMessage() {
  const qc = useQueryClient();
  const token = useAuthStore(s => s.token);
  useEffect(() => {
    if (!token) return;
    const s = getSocket(token);
    s.on('message:new', (msg) => {
      qc.setQueryData(['messages', msg.conversationId], (old: any) => ({
        ...old,
        items: [msg, ...(old?.items ?? [])],
      }));
      qc.invalidateQueries(['conversations']);
    });
    return () => { s.off('message:new'); };
  }, [token]);
}
```

## 11. Không được làm
